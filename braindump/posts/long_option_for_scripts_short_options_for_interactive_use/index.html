Long Option for Scripts, Short Options for Interactive Use

Fleeting

I think it helps a lot readability to make the options explicit.

Like all rules, it is meant to be thought about and not blindly followed. To me, it is a means to the more generic goal of game of communication.

For instance, I have some exceptions, like rm -rf or ls -la.

Also, there are some other tools for which I wonder if the pattern is known enough to use short options, like curl --fail --silent --show-error --location vs curl -fsSL (curl)
