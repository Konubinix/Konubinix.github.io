clk encourage de scripter par abstraction.

Notamment dans le clk flow. Chaque commande du flow doit être une abstraction du comportement.

Ainsi, le flow de commandes configure -> build est un flow abstrait.

configure peut être implanté par cmake, ou les autotools

build peut être implanté par ninja ou make.

Grâce aux clk alias, clk parameters, clk plugins ou clk extensions, l’implantation de chaque commande devient polymorphic.
